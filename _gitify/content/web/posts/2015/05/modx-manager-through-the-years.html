      Change is all around us, and the MODX manager is no exception. Throughout the years, the manager has seen a bunch of different styles, and it is still evolving. For #ThrowbackThursday, we look at how the MODX manager evolved over the past few years.

      We grabbed the same screenshot for each of the versions to provide some comparison material.

      1. The setup start screen
      2. The login screen
      3. The dashboard with default widgets
      4. The package browser home screen
      5. Package management with TinyMCE installed
      6. Editing the home resource with focus on the pagetitle
      7. Browsing an assets/uploads/ folder with 4 uploaded images, the first selected, using the media browser
      8. The system settings page
      9. Editing a user group.

      We repeated this cycle for MODX Revolution 2.0.8, 2.1.5, 2.2.15 and 2.3.3 - the most recent version in each of the minor release of Revolution to date. 

      MODX Revolution 2.0.8

      As the last release in the 2.0.x series, 2.0.8 does not exactly match the early days of Revolution. Looking at 2.0.0-beta1 for example, you can see it has progressed massively since the first inception. 

      MODX Revolution 2.1.5

      Visually speaking, 2.1.5 looks very similar to 2.0.8. The biggest difference is probably a new logo in the top right of the manager. 

      MODX Revolution 2.2.15

      In the 2.2 series, the manager got a big restyle, and it started shipping with a new logo again! One of the key improvements include package management and the package browser, which is no longer stuck in a small modal. The setup and login also got a much needed makeover.  

      MODX Revolution 2.3.3

      With 2.3.0, released July 16th 2014, again came a redesign of of the manager. Also a new logo iteration again :)

      For this article I couldn't get 2.0.0-pl installed due to MySQL issues, but if you want to see what MODX looked like before 2.0.8 (including Evolution!), be sure to leave a comment!
