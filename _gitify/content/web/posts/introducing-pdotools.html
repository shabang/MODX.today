And, what important, almost of them is a drop-in replacement for other popular snippets.\\n\\n#### pdoResources\\nIt is replacement for getResources.\\nAs all other snippets, it is not use xPDO objects for representation of resources. It works directly with database tables, so you do not need to convert resources dates to timestamps before pass them to \\\"date\\\" output filter.\\nAnd it joins only needed TVs, so you must specify them in `&includeTVs` parameter as comma-separated list.\\n```\\n[[!pdoResources?\\n &parents=`0`\\n &includeTVs=`my_tv1,my_tv2`\\n &tvPrefix=`tv.`\\n &processTVs=`1`\\n &includeContent=`1`\\n &showLog=`1`\\n]]\\n```\\n`&showLog` - is the one of the most important futures of pdoTools snippets. It allows you to controll how fast this snippet processing, exaclty. Is there a slow SQL query, or too complicated chunk? You will know.\\n\\n[![](https:\/\/file.modx.pro\/files\/d\/9\/3\/d931de6c3c2f67be8879c1765e833bbcs.jpg)](https:\/\/file.modx.pro\/files\/d\/9\/3\/d931de6c3c2f67be8879c1765e833bbc.png)\\n\\nThe second important future that you could see on screenshot - SQL joins. What the difference?\\n1. It do not shows empty pages. If you do not have results on page or user entered wrong page manually - you will get redirect to the first page.\\n2. It has parameter `&maxLimit` equal to 100 by default. User could not hang your site by setting `&limit=100000` in url. If you are using getPage - just try to do it on your site.\\n3. It sets metatags to previous and next pages for crawlers.\\n4. It has built-in ajax pagination, seriosly. You need only correct markup and one parameter:\\n```\\n
\\n
\\n [[!pdoPage?\\n &parents=`0`\\n &ajaxMode=`default`\\n ]]\\n <\/div>\\n [[!+page.nav]]\\n<\/div>\\n```\\nOf course, you can change markup by specifying additional parameters to snippet.\\n\\n#### pdoMenu\\nThis snippet has no so many difference with Wayfinder but works a little faster.\\n\\nBecause of bypass of xPDO objects you need to check permissions of items in menu by special parameter:\\n```\\n[[!pdoMenu?\\n &parents=`0`\\n &checkPermissions=`list`\\n]]\\n```\\nBy default, all permissions checks are disabled. This parameter common for all pdoTools snippets, by the way.\\n\\n#### pdoUsers\\nThis snippets lists users of your site. You can filter them by groups and roles:\\n```\\n[[!pdoUsers?\\n &groups=`Authors`\\n &sortdir=`asc`\\n]]\\n```\\nYou could combine it with **pdoPage**:\\n```\\n[[!pdoUsers?\\n &groups=`Authors`\\n &roles=`Member`\\n &sortby=`id`\\n &sortdir=`asc`\\n]]\\n[[!+page.nav]]\\n```\\nPlaceholders are `modUser` and `modUserProfile` fields. You can see them all if you will just set blank `&tpl` parameter (or do not set it at all, if snippet do not has default tpl value).\\n\\n[![](https:\/\/file.modx.pro\/files\/0\/a\/d\/0ad486c2c7412ad9a32c25f9027b3962s.jpg)](https:\/\/file.modx.pro\/files\/0\/a\/d\/0ad486c2c7412ad9a32c25f9027b3962.png)\\n\\nBlank `&tpl` is another common feauture of pdoTools snippets. You will not forget usable placeholders, right?\\n\\n#### pdoSitemap\\nFast sitemap that do not requires to set XML type of resource. If parameter `&forceXML` is enabled (by default).\\n\\nIt caches data by default and cache key depends from snippet params but you can set your own key. How fast it is?\\n\\nWell, we can test it by disabling `&forceXML` and enabling `&showLog`. On my site it takes almost 30 seconds for the first run and only 0.03 for the second:\\n```\\n[[!pdoSitemap?\\n &forceXML=`0`\\n &showLog=`1`\\n]]\\n```\\nThere are **6873** resources in my sitemap.\\n\\n[![](https:\/\/file.modx.pro\/files\/0\/d\/3\/0d3b7798fd8464ee2bb39fbb481e3902s.jpg)](https:\/\/file.modx.pro\/files\/0\/d\/3\/0d3b7798fd8464ee2bb39fbb481e3902.png) [![](https:\/\/file.modx.pro\/files\/1\/9\/1\/1916b6d00e0c7e77e0119c29f1d3aba6s.jpg)](https:\/\/file.modx.pro\/files\/1\/9\/1\/1916b6d00e0c7e77e0119c29f1d3aba6.png)\\n\\n### pdoNeighbors\\nSnippet for make links to previous, next and upper pages of current document.\\n```\\n[[!pdoNeighbors?\\n &sortby=`menuindex`\\n &sortdirc=`desc`\\n]]\\n```\\nJust neighbors, you know.\\n\\n#### pdoCrumbs\\nMy version of simple snippet for breadcrumbs of site. Nothing special except use of pdoTools core with fast chunks processing and data fetching.\\n```\\n[[!Breadcrumbs]]\\n```\\n\\n#### pdoTitle\\nThis snippet generates \\\"title\\\" tag of pages. It runs **pdoCrumbs** and shows path to current document in title.\\n```\\n[[!pdoTitle]] \/ [[++site_name]]<\/title>\\n```\\nAlso it supports **pdoPage** by default and you will see number of pages in title.\\n```\\n<title>Questions \/ page 5 from 593 \/ Sections \/ mysite.com<\/title>\\n```\\nCrawlers loves it!\\n\\n[![](https:\/\/file.modx.pro\/files\/a\/e\/f\/aef145845f8c84ad6bc104fe31e6d796s.jpg)](https:\/\/file.modx.pro\/files\/a\/e\/f\/aef145845f8c84ad6bc104fe31e6d796.png)\\n\\n#### pdoField\\nAnd, finally, snippet that allows you to get any field from any resources. All we have is here <a href="https://github.com/bezumkin/Docs/tree/master/en/01_Components/01_pdoTools">https://github.com/bezumkin/Docs/tree/master/en/01_Components/01_pdoTools</a>. You could add your own pages, by the way.</p> <p>And do not forget that all of snippets parameters are described inside them. Just see the appropriate tab in MODX manager.</p> </div> </div>
