MODX will serve up the resource as an XML file.</p> <h3>Elements</h3> <p>MODX Elements include Templates, Template Variables, Chunks, Snippets, and Plugins. To create a Template, click the plus icon next to ‘Templates’ in the ‘Elements’ tab.</p><p>The markup of a template is stored in the database by default. If you use version control such as Git, you’ll probably want to use static files. To do this, select the template you want to edit, then select ‘Is Static’ option and navigate to the relevant file.</p><p>Templates can’t contain PHP code; you’ll want to use Snippets for that. While at first this might seem frustrating, the separation of logic and presentation creates easy to read code.</p><p>Here’s what a very basic template in MODX looks like:</p> <pre><code class="language-markup"><html> <head> <title>[[++site_name]] - [[*pagetitle]]</title> <base href="[[++site_url]]" /> </head> <body> [[*content]] </body> </html></code></pre> <p>Nice and clean, huh? No emoji JavaScript, no meta tags, or any of the other cruft that WordPress throws in.</p> <h3>Template Variables</h3> <p>Template Variables or "TVs" are the equivalent to WordPress custom fields. A TV is tied to a template (or multiple templates).
