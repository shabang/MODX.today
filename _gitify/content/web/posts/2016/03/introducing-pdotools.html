You can see them all if you will just set blank <code>&tpl</code> parameter (or do not set it at all, if snippet does not have a default tpl value).</p> <figure class="content-image extended-width "> <img data-src="/assets/uploads/images/2016/03/pdotools3.png" class="lazyload" alt="" > </figure> <p>The empty <code>&tpl</code> is another common feature of all pdoTools snippets.</p> <h4>pdoSitemap</h4> <p>pdoSitemap creates a fast sitemap that does not require the resource to be set to XML if the property <code>&forceXML</code> is enabled (by default).</p> <p>It caches data, and the cache key depends on the snippet params, but you can also set your own key. </p> <p>To see how fast it is, we can test it by disabling <code>&forceXML</code> and enabling <code>&showLog</code>. On my site it takes almost 30 seconds for the first run and only 0.03 for the second:</p> <pre><code class="language-markup">[[!pdoSitemap? &forceXML=`0` &showLog=`1` ]]</code></pre> <p>There are <strong>6873</strong> resources in my sitemap.</p> <figure class="content-image extended-width "> <img data-src="/assets/uploads/images/2016/03/pdotools4.png" class="lazyload" alt="" > </figure> <figure class="content-image extended-width "> <img data-src="/assets/uploads/images/2016/03/pdotools5.png" class="lazyload" alt="" > </figure> <h3>pdoNeighbors</h3> <p>Snippet for make links to previous, next and upper pages of the current document.</p> <pre><code class="language-markup">[[!pdoNeighbors? &sortby=`menuindex` &sortdirc=`desc` ]]</code></pre> <h4>pdoCrumbs</h4> <p>My version of a simple snippet for breadcrumbs on the site. Nothing special, except it uses the pdoTools core with fast chunks processing and data fetching.</p> <pre><code class="language-markup">[[!pdoCrumbs]]</code></pre> <h4>pdoTitle</h4> <p>This snippet generates the "title" tag of pages. It runs <strong>pdoCrumbs</strong> and shows the path to the current document in the title.</p> <pre><code class="language-markup"><title>[[!pdoTitle]] / [[++site_name]]</title></code></pre> <p>It supports <strong>pdoPage</strong> by default, so you will see the number of pages in title.</p> <pre><code class="language-markup"><title>Questions / page 5 from 593 / Sections / mysite.com</title></code></pre> <p>Crawlers love it!</p> <figure class="content-image extended-width "> <img data-src="/assets/uploads/images/2016/03/pdotools6.png" class="lazyload" alt="" > </figure> <h4>pdoField</h4> <p>And, finally, pdoField is a snippet that allows you to get any field from any resources. It is a replacement both for <strong>UltimateParent</strong> and <strong>getResourceField</strong> snippets.</p> <p>For example, we need to get <code>longtitle</code> of resource with id = 15</p> <pre><code class="language-markup">[[pdoField? &id=`15` &field=`longtitle` ]]</code></pre> <p>Or if we want to get <code>pagetitle</code> of grandparent of current document:</p> <pre><code class="language-markup">[[pdoField? &id=`[[*id]]` &field=`pagetitle` &top=`2` ]]</code></pre> <h3>Conclusion</h3> <p>I can write more and more about pdoTools, but this is enough for the first article. In the next two articles I will talk about speed, and the pdoParser. </p> <p>It is important to remember, that all pdoTools snippets use the same core and most parameters can be used for all of them.</p> <p>You can use <code>&showLog</code> to see how the snippets work, you can specify empty chunks to see the available placeholders, and you can join tables on the fly.</p> <p>pdoTools is a library, not just set of snippets. Many MODX extras uses pdoTools for their own snippets, like miniShop2, Tickets, BannerY, AjaxForm, and more.</p> <p>The biggest drawback of pdoTools is the limited English documentation. All we have is here <a href="https://github.com/bezumkin/Docs/tree/master/en/01_Components/01_pdoTools">https://github.com/bezumkin/Docs/tree/master/en/01_Components/01_pdoTools</a>. If you're interested, you can help by adding more pages to the documentation.</p> <p>And don't forget that all of snippets parameters are described inside them. Just open the snippet in the manager and see the Properties tab.</p> </div> </div>
